How to use it
User guide

Five steps to your next lead

The tracker updates every morning. Here's how to get from the map to a real conversation.

Pick a category

The four toggle buttons filter everything — map and table together. Tap one to focus on that license type. Tap it again to show all.

On-Premise Off-Premise Wholesale Other

Search or narrow down

Type a business name, street, or city. Use the dropdown to filter by a specific license type — e.g. Type 47 (On-Sale General — Eating Place) for restaurants with full bars.

Tap a card or a pin

Each business shows as a card in the table and a dot on the map. Tap a card and the map zooms to that location. Tap a map pin to see the full details popup.

Get directions

Every card and popup has a Google Maps link. Tap it to get turn-by-turn directions or to look the business up before you call.

Make contact

When we have a phone number or website on file, it's right on the card. Tap the number to call, or the link to open their site. Contact data enriches automatically as new businesses appear.

First-mover advantage

New licenses appear in the tracker the same morning the CA ABC publishes them. That's often weeks before a new bar or restaurant opens its doors — time to build a relationship before anyone else is at the door.

Geographic intelligence

Every address is geocoded on first appearance. You see where new accounts are opening — by neighborhood, by zip, by city. Spot clusters, prioritize routes, identify territory gaps at a glance.

⚙️

Fully automated

The pipeline runs itself. No spreadsheet downloads, no manual filtering, no human in the loop. One person built it; nobody needs to maintain it. If a run fails, an alert fires automatically.

Rolling 90-day window

The tracker shows the last 90 days of new licenses — enough history to work from, recent enough to be actionable. The window advances daily so old data ages out automatically.

CA ABC Daily Export

Data source

The California Department of Alcoholic Beverage Control publishes a full statewide license export every day as a ZIP file (~7MB, ~27MB uncompressed CSV). It contains every active and pending license in the state. The pipeline pulls this directly — no API key required, no scraping, no fragile web automation.

abc.ca.gov/wp-content/uploads/DailyExport-CSV.zip
🔄

n8n Automation

Orchestration — self-hosted on Mac Mini M2

n8n is a self-hosted workflow automation tool (like Zapier, but runs locally). A scheduled trigger fires at 8am PT. The workflow downloads the ZIP, unzips it, parses the CSV, and filters to San Diego County licenses issued within the last 90 days. Roughly 100–130 records after deduplication. All processing happens in JavaScript code nodes — no external APIs for the core parse step.

Schedule: 08:00 PT daily — 8 nodes total
📍

Geocoding

Nominatim / OpenStreetMap

New businesses (ones not seen in the previous run) are geocoded using Nominatim — the free OpenStreetMap geocoding API. Street address, city, and zip are submitted; lat/lng comes back. Results are cached to disk inside the n8n container so each address is only ever geocoded once. Rate limited to 1 request per 1.1 seconds to stay within Nominatim's usage policy.

~11 new addresses geocoded on a typical run
📞

Contact Enrichment

Foursquare Places API

Businesses that have been geocoded but are missing a phone number or website are looked up in the Foursquare Places API using name + coordinates. Phone and website are written back into the record. Enrichment is additive — it never overwrites data that's already there. Currently 27 phone numbers and 22 websites on file across the dataset.

Foursquare Places v3 · radius: 200m
🐙

GitHub Commit

Version-controlled data layer

The final JSON (licenses array + metadata + timestamp) is base64-encoded and PUT directly to the GitHub API, updating a single file in the site's repository. This creates a named commit with every day's data snapshot — giving a complete audit trail of when each license appeared. The commit is the deployment trigger.

jordon/jordon-abc-data.json · GitHub Contents API
🌐

Cloudflare Pages

Hosting + global CDN

The entire front end is static HTML, CSS, and JavaScript — no server, no database, no runtime. Cloudflare Pages watches the GitHub repository and auto-deploys within ~30 seconds of any commit. The site is served from Cloudflare's global edge network. Zero hosting cost, sub-100ms load times worldwide.

roblogic.org → Cloudflare Pages → GitHub repo
⚠️

Error handling & alerting

If any step in the pipeline fails, n8n's error trigger fires and sends a push notification via ntfy. The previous day's data stays live on the site — visitors always see the last successful run. Failures are isolated: a geocoding timeout or Foursquare API error won't abort the commit; it just skips enrichment for that run and tries again tomorrow.
