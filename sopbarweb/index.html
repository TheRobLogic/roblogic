What the Code node does

  1. Filters events: only future events, removes past shows
  2. Removes weekly recurring: beer pong, karaoke, free pool, dart tourney/league (matched by title)
  3. Handles holds: "Hold" in title → converted to TBA. Fully-TBA cards hidden, partial like "Band / TBA" kept
  4. Parses descriptions: extracts doors, cover, links, flyer URLs from calendar event description fields (doors: cover: price: flyer: prefixes)
  5. Strips internal notes: removes booking notes (booked by, kayla, need to find, etc.)
  6. TickPick search: searches first band name, matches by date + venue name "bancroft", injects ticket URL
  7. Band links: fetches band-links.json via GitHub API, auto-links band names
  8. Festival titles: preserves prefix when bands come from description (e.g. "Whatever68 Fest: Band / Band")
  9. HTML generation: builds show cards for shows.html + compact list for homepage (next 5)
  10. Marker replacement: swaps content between <!-- SHOWS-START --> / <!-- SHOWS-END --> (and equivalent homepage markers)
  11. Diff summary: compares old vs new, reports added/removed shows in ntfy notification

■ Band Links

A JSON file maps band names to their social/music URLs. The n8n Code node fetches it from GitHub on every sync run and auto-links matching band names in show cards and homepage listings.

band-links.json
n8n Code node
Linked band names

File location: band-links.json in the repo root (TheRobLogic/bancroft).

Format:

{
  "Band Name": "https://instagram.com/bandname",
  "Other Band": "https://bandcamp.com/otherband"
}

To add links: Edit band-links.json, push to main. The next sync run (noon daily, or trigger the webhook) picks them up automatically. Links appear in both /shows cards and the homepage "Coming Up" section.

The file is fetched via the GitHub Contents API (not raw URL) because the repo is private.

■ Credentials Inventory

Expiring credentials

Automated ntfy reminders are set up via crontab. You'll get a heads-up one month before, then weekly nags in the final month.

CredentialWhereExpiresWhat breaks
GitHub PAT n8n-bancroft-site n8n HTTP headers Mar 1, 2027 n8n can't push show updates to GitHub
TickPick JWT n8n Code node ~Sep 3, 2026 Ticket links stop appearing (shows still work)
Google Calendar OAuth2 n8n credential store Auto-refreshes Calendar read stops (rare — OAuth handles refresh)
Resend API key CF Worker secret No expiry Booking form emails stop sending
Cloudflare account wrangler login (local) Session-based Can't deploy Worker updates (site itself unaffected)

Renewal procedures

  • GitHub PAT: GitHub → Settings → Developer settings → Fine-grained tokens. Create new token with Contents R/W on TheRobLogic/bancroft + roblogic repos. Update in n8n HTTP Request node headers.
  • TickPick JWT: Log into tickpick.com/organizer, extract new JWT from browser network tab (look for x-user-token header). Update in n8n Code node.

Reminder system

Crontab runs ~/scripts/bancroft-reminders.sh every Monday at 9am. It checks the current month/year and sends ntfy push notifications when inside a reminder window. Silent the rest of the time.

■ Common Operations

Add a show

Add an event to the SHOWS Google Calendar. Title format: Band 1 / Band 2 / Band 3. Optional description lines for doors, cover, flyer URL. The next daily sync (noon) or a manual webhook trigger picks it up.

Add band links

Edit band-links.json in the repo. Add "Band Name": "https://url". Push to main. Next sync auto-links them.

Trigger a manual sync

curl https://n8n.roblogic.org/webhook/bancroft-shows-sync

Returns a JSON ntfy response with the sync summary.

Mark a show as "hold" or TBA

Put "Hold" in the calendar event title. The code converts it to TBA. A fully-TBA card is hidden from the site. If it's partial like Band Name / Hold, it shows as "Band Name / TBA".

Add a festival / multi-day event

Set the event title to the festival name (e.g. "Whatever68 Fest"). List individual bands in the description, one per line under a Bands: header. The output will be "Whatever68 Fest: Band1 / Band2 / Band3".

Update the booking form recipient

The recipient email is hardcoded in the Cloudflare Worker. To change it:

# In ~/github/bancroft-form-worker/
# Edit src/index.js, change the "to" address
wrangler deploy

Update beer menu

The On Tap page is an Untappd embed. It updates automatically from Untappd. No action needed.

Edit the site directly

cd ~/github/bancroft-prod
# Make changes
git add -A && git commit -m "description" && git push
# Cloudflare auto-deploys in ~30 seconds
Note

Manual edits to shows.html or the homepage shows section will be overwritten by the next n8n sync. Only edit content outside the <!-- SHOWS-START --> / <!-- SHOWS-END --> markers.
